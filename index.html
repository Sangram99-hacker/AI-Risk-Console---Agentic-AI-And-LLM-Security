Why this exists
Two frameworks, one attack surface

LLM applications and agentic AI systems fail in ways traditional AppSec checklists don't cover. This console pairs the industry-standard OWASP Top 10 for LLM Applications (2025) — the risks inside a single model's inputs, outputs, and data — with the MAESTRO framework from the Cloud Security Alliance, which threat-models the full stack an autonomous agent runs on, from the foundation model up to the multi-agent ecosystem it operates in.

🛡️ OWASP Top 10 for LLM Applications

Ranks the most critical risks observed in production LLM applications — prompt injection, data leakage, supply chain compromise, and more. Best for: chatbots, copilots, RAG systems, anything a model directly powers.

🧬 MAESTRO Agentic Threat Model

Multi-Agent Environment, Security, Threat, Risk & Outcome. A seven-layer decomposition built for systems where agents plan, use tools, and act autonomously — including multi-agent collusion and identity attacks. Best for: autonomous agents, tool-using systems, multi-agent orchestration.

Cross-layer threats
When one layer's breach becomes another layer's problem

A compromise rarely stays in its lane. An attacker who breaches container infrastructure (L4) can pivot into the agent's data store (L2), poisoning the next model fine-tune (L1) — cascading up through every layer above it.
